# Информационная система «Сервис заявок на ремонт техники»

Проект выполнен в рамках производственной практики по профессиональному модулю ПМ.05 «Проектирование и разработка информационных систем».

## Описание проекта

Информационная система предназначена для оформления и обработки заявок на ремонт техники. Пользователь может зарегистрироваться, авторизоваться, создать заявку на ремонт и просмотреть список своих заявок. Администратор может просматривать все заявки пользователей и изменять их статус.

## Предметная область

Предметная область проекта — сервис ремонта техники.

Основные процессы:

* регистрация пользователя;
* авторизация пользователя;
* создание заявки на ремонт;
* просмотр списка заявок;
* изменение статуса заявки администратором.

## Сущности базы данных

### users

Таблица хранит данные пользователей системы.

Поля:

* id — первичный ключ;
* login — логин пользователя;
* password_hash — хэш пароля;
* full_name — ФИО;
* phone — телефон;
* email — адрес электронной почты;
* role — роль пользователя;
* created_at — дата создания записи.

### statuses

Таблица хранит статусы заявок.

Поля:

* id — первичный ключ;
* name — название статуса.

Используемые статусы:

* Новая;
* В работе;
* Завершена.

### requests

Таблица хранит заявки на ремонт техники.

Поля:

* id — первичный ключ;
* user_id — внешний ключ на пользователя;
* status_id — внешний ключ на статус;
* device_type — тип техники;
* problem_description — описание проблемы;
* contact_method — способ связи;
* created_at — дата создания заявки.

## Основные сценарии работы

### Регистрация

Пользователь заполняет форму регистрации: логин, пароль, ФИО, телефон и e-mail. После проверки данных запись пользователя сохраняется в базе данных.

### Авторизация

Пользователь вводит логин и пароль. При успешной проверке пользователь переходит в личный кабинет. Администратор после входа переходит в панель администратора.

### Работа с заявками

Пользователь в личном кабинете создает заявку на ремонт техники и просматривает список своих заявок.

### Панель администратора

Администратор просматривает все заявки пользователей и изменяет их статус.

## Запуск проекта

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Создать базу данных MySQL с помощью скрипта:

```text
sql/schema.sql
```

3. Запустить приложение:

```bash
python app.py
```

4. Открыть приложение в браузере:

```text
http://127.0.0.1:5000/
```

5. Создать администратора:

```text
http://127.0.0.1:5000/create-admin
```

Тестовые данные администратора:

```text
Логин: admin
Пароль: admin12345
```
